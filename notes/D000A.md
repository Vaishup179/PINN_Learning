#### Activation, Bias and Weights
> Each neuron have numbers which known as activation. Those digits are kind of data or information. They pass the information. To pass such information there are layers of neuron there. Layers are essentially the 'stage' of processing that information goes through to reach a conclusion.

So, Neural networks have "INPUT LAYER", "HIDDEN LAYERS", "OUTPUT LAYER". 

    1. "INPUT LAYER" is entry point, from where inital information comes. 
    2. "OUTPUT LAYER" is the final stage of neuron who provides the answers
    3. "HIDDEN LAYER" is a collection of neurons that take inputs from previous layer - multiply by some digits, and add something some-values and provides a result. 

In the HIDDEN LAYER", when I used the words - multiply by some digits are actually the weights and some-values which have been added as - Bias.
Basically, the "INPUT LATERS", "OUTPUTLAYERS" AND "HIDDEN LAYERS" are connected with some networks. TO make such kinds of network, Weights and Biases are required. 
In pure technical terms,

> **ACTIVATION** -------> Information or state of a neuron
> **WEIGHTS**    -------> The conncetions
> **BIAS**       -------> Threshold shifter

##### WEIGHTS :     
- Each connection between neurons are weights. It is a number attached to every conncetion between neurons.
- It tells the network how much to "listen" to a specific piece of information.
- If a weight is large and positive, it turns the volume up on that pixel because it's important for a pattern. 
- If it's negative, it essentially says, if this pixel is bright, its probably not the digit i'm looking for, and turns the signal down.
- We everytime mupltiply each input by its weights. (input * weight)

##### BIAS :   
- It is like a silecne setting shifter for each neuron.
- It is an extra number added to the total sum of the weighted inputs.
- It determines how much total signal is needed before the neuron is allowed to make any sound at all.
- Even if the weights pick up some signals, a negative bias acts as a hurdle. The signal must be louder than 10 to overcome the bias and move forward.
- High bias value makes it very easy for the neuron to activate.
- When low or negative bias values commes, the neuron becomes "stubborn". Even if the inputs are strong, the weighted sum has to be very high to overcome a negative bias.

Bias and Weights togather have an equation as : 
```z = w₁x₁ + w₂x₂ + ... + b```

##### ACTIVATION : 
- Input, wights and bias follows some rule by an function, which is Activation function.
- The activation function is the mathematical rule that takes the final result of (Weights * Input) + Bias and turns it into the final activation value.
- Different functions result in different shapes :
    - Sigmoid : A "S-Shaped" curve that keeps the activation strictly between 0 and 1. This is classic way to reprents a neuron's intensity.
    - ReLU (Rectified Linear Unit) : A hockey - stick graph. If number is negative, it become 0. If number is positive, it stays exactly as it is or higher than 1. 
    - tanh : An "S-shaped" curve like sigmoid but output lies between -1 and 1 instead of 0 and 1.This is the preferred activation function in PINN!
- **Why do we need Activation Functions at all?**
Without them, no matter how many hidden layers we stack, the entire network collapses into a single straight line — only capable of learning linear patterns. Activation functions introduce 
non-linearity, allowing the network to learn complex curved patterns like handwritten digits,faces, or XRD diffraction peaks.

#### Connection to PINN Research (Generalized)

Neural Networks are the **core engine** of PINN.
In any PINN application, the neural network 
replaces the traditional numerical solver.

**General PINN Structure:**

| Component | Traditional Solver | PINN |
|---|---|---|
| Input neurons | Grid points (x, t) | Coordinates (x, t) or any domain variable |
| Hidden layers | Computation steps | Learning the physics pattern |
| Output neuron | u(x,t) from FEM/FDM | u(x,t) approximated by network |
| Weights & Biases | Not applicable | Adjusted during training |
| Constraint | Boundary conditions only | Data + PDE + Boundary conditions |

---

**General Input → Output Examples Across Domains:**

| Domain | Input Neurons | Output Neuron |
|---|---|---|
| Heat Transfer | position x, time t | temperature u(x,t) |
| Fluid Mechanics | x, y, t | velocity, pressure |
| Wave Propagation | x, t | wave amplitude u(x,t) |
| Materials (XRD) | diffraction angle 2θ | crystallite size D |
| Medical Imaging | pixel values | tissue type / diagnosis |
| Earthquake | seismic signal values | ground motion prediction |
| Drug Discovery | molecular structure | binding energy |

---

**The Universal PINN Statement:**
> Given ANY physical system governed by a PDE,
> a neural network can be trained to approximate 
> its solution by minimizing:
>
> **L = L_data + L_pde + L_boundary**
>
> Where:
> - L_data = how well it fits experimental measurements
> - L_pde = how well it satisfies the physics equation
> - L_boundary = how well it satisfies boundary conditions

---

**Why Neural Network is the Right Tool for PINN:**
- It is a **universal function approximator** 
  (Universal Approximation Theorem)
- It can handle **irregular domains** and 
  **noisy experimental data**
- It naturally incorporates **physics constraints** 
  into its loss function
- It works even when **data is scarce** — 
  physics fills the gaps

---

#### Universal Approximation Theorem

A neural network with sufficient neurons can 
approximate **any** continuous mathematical function
to any desired accuracy.

**What this means for PINN generally:**
Any PDE solution u(x,t) that is a continuous 
function CAN be represented by a neural network.
This is the mathematical guarantee that makes 
PINN theoretically valid — not just an experiment,
but a mathematically justified approach.
